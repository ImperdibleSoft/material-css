Floating windows are a good way to have some content displayed permanently on the screen, while the user navigates between sections.

They are useful to show adds, chats, even multimedia content. So we have prepeared this module to be used with our framework.

Common windows

First of all, we have to set the common styles with the "mc-floating-window" class.

This content is prepeared to have one "mc-window-header", one "mc-window-content" and one "mc-window-footer", and it will be displayed on the bottom right corner.

  • <div class="mc-floating-window" >
    • <div class="mc-window-header" >
      • <h1 class="mc-title" ></h1>
      • <h2 class="mc-note" ></h2>
    • </div>
    • <div class="mc-window-content" >
      • <p class="mc-text" ></p>
    • </div>
    • <div class="mc-window-footer" >
      • <p class="mc-note" ></p>
    • </div>
  • </div>

This module have to be placed before the mc-content, and you only can set 1 floating window at time.

Chat windows

Of course, we have set a few layouts to modify the floating windows' appearance. By using the mc-layout="chat" attribute on the "mc-floating-window" element, you will have available some special content for it.

On your "mc-window-content"

  • You will be able to set as "mc-message" elements as you want. This messages allows you to place an image and a "mc-text" into it.
  • You also can add the "mc-own" class to an "mc-message" element, so it'll display differently to indicate you have written this message.

On your "mc-window-footer"

  • You can add an <input type="text" /> and a "mc-button-icon".


  • <div class="mc-floating-window" mc-layout="chat" >
    • <div class="mc-window-header" > ... </div>
    • <div class="mc-window-content" >
      • <div class="mc-window-message" >
        • <img src="" />
        • <p class="mc-text" >...</p>
      • </div>
      • <div class="mc-window-message mc-own" >
        • <img src="" />
        • <p class="mc-text" >...</p>
      • </div>
    • </div>
    • <div class="mc-window-footer" >
      • <input type="text" />
      • <button class="mc-button mc-button-icon" mc-action="send"></button>
    • </div>
  • </div>

Multimedia windows

Multimedia players used to be a hard component to design and develop. That's why we have paid so much attention to this component. By using the mc-layout="multimedia" attribute on the "mc-floating-window" element, you will have available some special content for it.

On your "mc-window-content"

  • you will be able to set an "img" element, that will occupy all available space to show the cover of the album/track.
  • Another option is to place an mc-layout="list" attribute on the "mc-window-content", and create a list into it, so you will be able to display a beautifull playlist.

On your "mc-window-footer"

  • You will need to set a "mc-slider" to modify the timming of the track (advance or go back).
  • You also will need to set a "audio" tag, to reproduce the track.
  • Finally, you can place between 2 and 6 "mc-button-icon" to display your controls.


  • <div class="mc-floating-window" mc-layout="multimedia" >
    • <div class="mc-window-header" > ... </div>
    • <div class="mc-window-content" mc-layout="list">
      • <div class="mc-list-item" >
        • <p class="mc-title" >...</p>
        • <p class="mc-note" >...</p>
      • </div>
      • <div class="mc-list-item" > ... </div>
    • </div>
    • <div class="mc-window-footer" >
      • <div class="mc-slider" > ... </div>
      • <audio src=""></audio>
      • <button class="mc-button mc-button-icon" mc-action="music-prev"></button>
      • <button class="mc-button mc-button-icon" mc-action="play"></button>
      • <button class="mc-button mc-button-icon" mc-action="music-next"></button>
    • </div>
  • </div>

Share windows

A new and very common interaction on modern applications is to share the content. So we have enabled an special layout for this purpose.

Using the mc-layout="share" attribute, you will get an special Floating window. You can place whatever you want, with these particularities in mind.

  • mc-button-icons" are bigger than normal.
  • This window doesn't have "mc-window-footer".


  • <div class="mc-floating-window" mc-layout="share" >
    • <div class="mc-window-header" > ... </div>
    • <div class="mc-window-content">
      • <p class="mc-text" >...</p>
      • <div class="mc-input" >...</div>
      • <button class="mc-button mc-button-icon" mc-action=""></button>
      • <button class="mc-button mc-button-icon" mc-action=""></button>
      • <button class="mc-button mc-button-icon" mc-action=""></button>
    • </div>
  • </div>
